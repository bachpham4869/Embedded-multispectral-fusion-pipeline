## 5. Caveats Inserted

To keep claims strictly within their evidence boundaries, the following qualifiers were added to the manuscript:
- **Manual labels qualification:** Expressly noted that manual labels represent a visual review and correction process gaged by the user and agent, rather than independent gold-standard annotations.
- **BU-TIV surrogate status:** Clarified that BU-TIV is an external surrogate benchmark down-scaled to $80 \times 62$, and not native MI48 raw radiometric validation. Explicitly noted the order-mapping assumption (`FRAME_MAPPING_ASSUMPTION_order_mapping`).
- **Rule fallback simulation caveat:** Framed the no-retrain policy performance as an offline simulation from feature logs rather than a runtime integrate-and-test deployment.
- **Top-2 interpretation:** Explained that while exact 9-class accuracy is low due to environmental overlap, the high bucket-level accuracy ($85.7\%$) is the most relevant metric for adaptive processing dispatch.

## 7. Remaining Risks

1. **Lack of native MI48 anomaly labels:** Although MAD was tested on BU-TIV, validation on native MI48 thermal sequences remains pending.
2. **Offline simulation gap:** The rule fallback policy is simulated on offline feature logs. A live target-hardware integration check is required to verify real-time performance.
3. **Low-support classes:** Adverse conditions (rain, fog, transition) still lack sufficient labeled sensor-domain representation.
4. **Model latency checks:** Stage/session timing has been profiled, but standalone RF100/RF200 feature+predict latency checks on target hardware remain open and separate from full mode-matrix acceptance.
5. **No-reference night metrics:** BRISQUE remains a weak proxy for night-vision validation, requiring a customized night-vision no-reference quality metric in future work.
